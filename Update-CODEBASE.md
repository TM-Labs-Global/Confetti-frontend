# Confette вЂ” Codebase Structure

> **Stack:** Next.js 15 (App Router) В· React 19 В· TypeScript В· Tailwind CSS v4 В· Prisma 5 (SQLite) В· jose (JWT) В· lucide-react

---

## Overview

Confette is an **AI-assisted event planning and vendor marketplace** platform built for the Nigerian market. It is a **full-stack Next.js application** вЂ” the frontend (React pages and components) and the backend (API routes, database, auth) all live in this single repository, written entirely in **TypeScript**.

There are three distinct user roles, each with its own isolated portal:

| Role | Portal path | Responsibility |
|---|---|---|
| **Organiser** | `/organiser` | Creates event plans, allocates budgets, reviews vendor bids |
| **Vendor** | `/vendor` | Browses the marketplace, submits bids on plan categories |
| **Admin** | `/admin` | Oversees all plans, escrow accounts, and platform activity |

#### Portal Layouts вЂ” Auth Guard Pattern

Each portal layout (`organiser/layout.tsx`, `vendor/layout.tsx`, `admin/layout.tsx`) follows the same TypeScript pattern:

```tsx
// Example: src/app/organiser/layout.tsx
"use client";

import { useAuth } from "@/features/auth/context/AuthContext";
import { OrgSidebar } from "@/features/organiser/components/OrgSidebar";
import { TopBar } from "@/features/organiser/components/TopBar";
import { redirect } from "next/navigation";

export default function OrganiserLayout({ children }: { children: React.ReactNode }) {
  const { user, loading } = useAuth();

  if (loading) return <LoadingScreen />;
  if (!user) redirect("/login");
  if (user.role !== "organiser") redirect(`/${user.role}/dashboard`);

  return (
    <div className="shell">
      <OrgSidebar />
      <main>
        <TopBar />
        {children}
      </main>
    </div>
  );
}
```
