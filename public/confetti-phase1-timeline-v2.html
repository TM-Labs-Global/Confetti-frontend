Correction from v1.0 — AI categories removed
The original Phase 1 Timeline listed "AI Category Suggestions" as a feature and required Code to return ≥8 categories within 5 seconds. This conflicts with PRD v1.1 §1.4, which explicitly defers AI category generation from Phase 1 due to token cost and scalability concerns.

Phase 1 ships with pre-populated, Admin-managed vendor categories. The Admin Dashboard manages all event types and category lists, and changes are reflected instantly in the User App without a code deployment. AI-assisted suggestions are a candidate for a future phase.
Overview

Phase 1 at a Glance

Phase 1 delivers the complete foundational product layer across all three surfaces — User App, Vendor Dashboard, and Admin Dashboard — structured over four weeks. The frontend scaffold is already in place. Development begins at Day 4.

Days 1–3 · Complete
Setup & Infrastructure
Design system, frontend scaffold, brand tokens, routing, auth context, layouts, placeholder pages, GitHub repo.
Days 4–14 · Active
Core Feature Development
Auth, plan creation wizard, pre-populated categories, budget allocation, save/share, vendor marketplace, bid flow, Admin views & category management, notification bell.
Weeks 3–4
Hosting, Testing & QA
Staging deployment, end-to-end testing across all three surfaces, responsiveness QA, performance, bug fixes, acceptance sign-off.
Note on infrastructure tasks
DevOps (hosting, CI/CD, domain, environments) and legal docs (Terms of Service, Privacy Policy, Escrow Policy) run in parallel with feature development. These are not frontend tasks but are required before staging deployment in Weeks 3–4.
Days 4–14

Core Feature Development

● In Progress

All three surfaces are built in parallel. The goal is a working end-to-end prototype in staging — plan creation through to bid submission — without live payments.

Authentication · All Surfaces
FeatureDescriptionRole
Sign Up & Login Email/password registration with email verification. OAuth via Google and Facebook — role selection (Organiser or Vendor) prompted on first OAuth login. All roles
Email Verification Time-limited verification link. Unverified accounts cannot create plans, bid, or transact. All roles
Session Management JWT access tokens and refresh tokens. Brute-force protection on login. Optional 'Remember me' persistent session. All roles
Forgot Password Secure, time-limited reset link sent to the registered email. All roles
Role-Based Redirect After login, users are automatically routed to the correct product surface. Admin role is assigned by the backend only — not selectable on signup. All roles
Profile & Account Update display name, profile photo, and phone number. Change password. Delete account — guarded, no active plans or open escrow allowed. User
User App · Plan Creation Wizard
FeatureDescriptionRole
Step 1 — Plan Details Event type selector (Wedding, Birthday, Corporate, Baby Shower, Graduation, Anniversary, or custom type). Total budget in ₦. Four-field location entry (Country, State, City, Address). Date and time picker. Plan name. User
Step 2 — Category Selection Pre-populated set of 8–10 vendor categories loads automatically based on the selected event type. Categories are Admin-managed (see Admin Dashboard section). User can toggle categories on/off — at least one must remain selected — and add custom categories (e.g. Tent Hire, Photo Booth). Custom categories are labelled and removable at any time. User
Step 3 — Budget Allocation Manual allocation: enter exact amount per category. Smart Split: one-click proportional redistribution using each category's pre-set suggested budget percentage. Even Split: distributes total budget equally across all selected categories. Live progress bar tracks allocated vs. remaining. Real-time over-budget alert with exact overspend shown. User
Step 4 — Summary & Review Ranked visual breakdown with bar charts, percentages, allocated amounts, and remaining balance. Step navigation — click any step indicator to return without losing data. User
User App · Saving & Sharing Plans
FeatureDescriptionRole
Save Plan Save at any wizard step. Persists across sessions. Private by default. User
My Plans Dashboard List all saved plans with event type, total budget, category count, and last-saved date. Reopen or delete plans. User
Generate Share Code Publish a plan to shared storage with a unique 7-character code. Accessible to anyone with the code. User
Load Shared Plan Enter a share code to view a plan and use it as an editable personal template. All roles
Vendor Dashboard
FeatureDescriptionRole
Vendor Registration Business name, primary service category or categories, operating region (State / City), bank account details for payouts, and contact information. Bank details held for Admin verification before first payout. Vendor
Marketplace Feed All plans published via 'Find Vendors', filtered automatically by the vendor's registered service category and operating region. Vendors only see plans relevant to them. Vendor
Plan Detail View Event type, city and state, event date, per-category budget allocation, and Organiser display name. Full contact details withheld until escrow is confirmed. Vendor
New Plan Notification Email and in-app notification when a new matching plan is published. Vendor
Submit a Bid Bid amount and pitch message. Vendors can preview before submitting. One active bid per plan — prevents spam. Vendor
Counter-Bid Bid above the Organiser's allocated amount with a written justification. Vendor
Update Bid Revise bid amount and pitch once before the Organiser accepts. Vendor
My Bids All submitted bids with status: Pending · Accepted · Awaiting Escrow · Escrow Confirmed · In Progress · Completed · Rejected. Vendor
Admin Dashboard
FeatureDescriptionRole
All Plans View Every plan on the platform with filters for status (Draft · Open · Bidding · Escrow Pending · In Progress · Completed · Disputed), event type, date, and location. Admin
Plan Detail Full view: budget breakdown, categories, location, all bids, selected vendors, and escrow status. Admin
Submit Bid as Vendor Admins may submit bids on behalf of any registered vendor on any open plan. Admin
Flag / Remove Plans Remove inappropriate plans with a notification and reason sent to the Organiser. Admin
Manage Event Types Create, edit, rename, or deactivate event types (e.g. Wedding, Birthday, Corporate) available to Organisers in the plan creation wizard. Admin
Manage Vendor Categories Create, edit, rename, or deactivate vendor categories — including each category's default suggested budget percentage — per event type. Changes take effect immediately in the User App without a deployment. Supports unlimited categories. Admin
Notifications · Phase 1 Scope
FeatureDescriptionRole
New Open Plan Alert Vendors emailed and notified in-app when a new matching plan is published. Email delivered within 60 seconds of the event. System
Bid Submitted Organiser notified each time a vendor submits or updates a bid on their plan. System
In-App Bell Real-time bell icon in the topbar navigation across all three surfaces. Unread badge count. Notification feed for all key events. All roles
Phase 1 Development Milestone
A working prototype in staging where an Organiser can create a plan from pre-populated categories, a Vendor can browse matching plans and submit a bid, and an Admin can manage categories and view/manage both — without live payments.
